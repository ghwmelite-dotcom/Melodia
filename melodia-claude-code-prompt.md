# MELODIA — AI Music Generation Platform
## Claude Code Development Prompt

> **Project Owner:** Hodges & Co. Limited (Sub-brand: Melodia)
> **Stack:** Cloudflare-native (Workers, Pages, D1, R2, KV, Vectorize, Durable Objects, AI Gateway, Queues) + ACE-Step 1.5 on external GPU
> **Goal:** Build an AI-powered music platform that generates world-class, award-winning-level songs with full vocals and instrumentation from minimal user input.

---

## 1. PROJECT OVERVIEW

Melodia is an AI music generation platform that takes a user from a simple mood/idea (as little as "Afrobeats love song about Accra at night") to a fully produced, radio-ready song with lyrics, vocals, instrumentation, and album artwork — all generated by AI.

### Core Principle
The platform should feel like having a Grammy-winning songwriter, a world-class producer, and a professional graphic designer on call 24/7. The user provides the spark — the AI does everything else.

### Architecture Summary
```
User Input → Cloudflare Worker (orchestrator)
  ├── Step 1: LLM Lyrics Engine (Workers AI — Llama 4 Scout / Qwen3)
  ├── Step 2: Music Generation (ACE-Step 1.5 — External GPU via REST API)
  ├── Step 3: Album Artwork (Workers AI — FLUX.2 / Leonardo Lucid Origin)
  ├── Step 4: Post-Processing (Workers — metadata, waveform, storage)
  └── Step 5: Delivery (R2 → CDN → User with real-time progress via Durable Objects)
```

## 2. TECHNOLOGY STACK

### 2.1 Cloudflare Services (Primary Stack)

| Service | Purpose |
|---------|---------|
| **Cloudflare Pages** | React frontend (SPA) — song studio, library, community, player |
| **Cloudflare Workers** | API layer — routing, auth, orchestration, rate limiting |
| **Workers AI** | LLM for lyrics (`@cf/meta/llama-4-scout-17b-16e-instruct` or `@cf/qwen/qwen3-30b-a3b-fp8`), TTS (`@cf/deepgram/aura-1`), STT (`@cf/openai/whisper`), Image gen (`@cf/black-forest-labs/flux-2-dev` or `@cf/leonardo/lucid-origin`) |
| **AI Gateway** | Centralized logging, caching, rate limiting, fallback routing for all AI model calls |
| **D1** | SQLite database — users, songs, playlists, credits, generation history |
| **R2** | Object storage — audio files (WAV/MP3), stems, album artwork, waveform data |
| **KV** | Key-value cache — sessions, feature flags, user preferences, generation queue status |
| **Vectorize** | Vector database — song style embeddings for "similar songs" recommendations |
| **Durable Objects** | Stateful song generation sessions — real-time progress updates via WebSockets |
| **Queues** | Background job processing — batch generation, post-processing tasks |

### 2.2 External Service (Music Generation)

| Service | Purpose |
|---------|---------|
| **ACE-Step 1.5** | Open-source music generation model deployed on external GPU server (Vast.ai / RunPod / self-hosted). Exposes REST API at `http://<gpu-server>:8001`. Generates full songs with vocals, instrumentation, and structure from lyrics + style tags. |

## 5. SONG GENERATION PIPELINE (The Core Engine)

This is the most critical piece. The pipeline is orchestrated by a Cloudflare Durable Object that maintains state and pushes real-time updates via WebSocket.

### 5.1 Pipeline Stages

```
STAGE 1: INTAKE & EXPANSION (Workers AI — LLM)
  Input:  User prompt (e.g., "Afrobeats love song about Accra at night")
  Action: LLM expands minimal input into a complete song blueprint
  Output: {
    title: "Accra Nights",
    genre: "afrobeats",
    sub_genre: "afro-soul",
    mood: "romantic, nostalgic, warm",
    bpm: 95,
    key: "Dm",
    time_signature: "4/4",
    duration: 210,
    vocal_style: "smooth male vocal, slight rasp, soulful delivery",
    vocal_language: "en",
    instruments: ["african percussion", "guitar", "synth pad", "bass", "shaker"],
    style_tags: "afrobeats, afro-soul, romantic, night vibes, smooth vocal, guitar-driven, percussive, warm synth pads, Lagos-style production, danceable",
    artwork_mood: "nighttime cityscape, warm golden lights, African city, romantic atmosphere"
  }

STAGE 2: LYRICS GENERATION (Workers AI — LLM)
  Input:  Song blueprint from Stage 1
  Action: LLM generates world-class structured lyrics
  Output: Full lyrics with section markers:
    [Intro]
    ...
    [Verse 1]
    ...
    [Pre-Chorus]
    ...
    [Chorus]
    ...
    [Verse 2]
    ...
    [Bridge]
    ...
    [Chorus]
    ...
    [Outro]
    ...

STAGE 3: LYRICS REFINEMENT (Workers AI — LLM, optional self-critique loop)
  Input:  Generated lyrics + song blueprint
  Action: LLM reviews lyrics for quality (rhyme scheme, emotional arc, hook strength,
          syllable flow, genre authenticity) and refines
  Output: Polished final lyrics

STAGE 4: MUSIC GENERATION (ACE-Step 1.5 — External GPU)
  Input:  {
    tags: style_tags from blueprint (comma-separated),
    lyrics: structured lyrics with section markers,
    duration: target duration in seconds,
    seed: random or user-specified,
    num_variations: 4 (generate multiple takes)
  }
  Action: ACE-Step 1.5 REST API generates full songs with vocals + instrumentation
  Output: Array of audio files (WAV format, 48kHz stereo)

STAGE 5: ALBUM ARTWORK GENERATION (Workers AI — Image Model)
  Input:  Artwork prompt derived from song blueprint + title + mood
  Action: FLUX.2 or Leonardo generates album cover artwork
  Output: 1024x1024 PNG image

STAGE 6: POST-PROCESSING (Workers)
  Actions:
    - Upload all audio variations to R2
    - Upload artwork to R2
    - Generate waveform visualization data from audio
    - Extract duration and audio metadata
    - Store all metadata in D1
    - Generate song embeddings via Vectorize for recommendations
  Output: Completed song record with all URLs populated

STAGE 7: DELIVERY
  Actions:
    - Push final status via WebSocket
    - Song available for playback, download, sharing
```

### 5.2 LLM System Prompts

#### Stage 1: Song Blueprint Generator

```
SYSTEM PROMPT — SONG BLUEPRINT GENERATOR

You are the world's most accomplished music director and A&R executive. You've discovered and produced hundreds of award-winning songs across every genre. Your job is to take a user's raw idea — which could be as vague as a single word or as detailed as a full brief — and craft a comprehensive song blueprint that would guide a world-class production team.

RULES:
1. Always output valid JSON matching the schema below.
2. If the user gives minimal input, make BOLD creative choices. Don't play it safe — choose specific, evocative directions.
3. Match the genre authentically. An Afrobeats song needs West African percussion and rhythmic patterns. A country song needs steel guitar and storytelling. A trap song needs 808s and hi-hats.
4. BPM must be genre-appropriate (e.g., Afrobeats: 90-110, Trap: 130-160, Ballad: 60-80, Pop: 100-130, EDM: 120-150).
5. Key signature should match the mood (minor keys for melancholy/intensity, major for upbeat/happy).
6. Style tags should be richly descriptive — at least 10 comma-separated tags covering genre, sub-genre, mood, vocal style, instrumentation character, and production style.
7. Duration should be genre-appropriate (Pop: 180-240s, Ballad: 240-300s, EDM: 180-360s, Hip-hop: 180-240s).
8. Artwork mood should paint a vivid visual scene that captures the song's emotional essence.

OUTPUT SCHEMA:
{
  "title": "string — creative, memorable song title",
  "genre": "string — primary genre",
  "sub_genre": "string — specific sub-genre",
  "mood": "string — 2-4 mood descriptors, comma-separated",
  "bpm": number,
  "key": "string — e.g., 'Dm', 'G', 'Bb minor'",
  "time_signature": "string — e.g., '4/4', '3/4', '6/8'",
  "duration": number (seconds),
  "vocal_style": "string — detailed vocal character description",
  "vocal_language": "string — ISO 639-1 code",
  "instruments": ["array", "of", "specific", "instruments"],
  "style_tags": "string — richly descriptive comma-separated tags for ACE-Step",
  "artwork_mood": "string — vivid visual scene description for image generation",
  "song_concept": "string — 2-3 sentence creative direction for the lyrics"
}
```

#### Stage 2: Lyrics Generator

```
SYSTEM PROMPT — WORLD-CLASS SONGWRITER

You are a legendary songwriter who has penned #1 hits across every genre — from Afrobeats anthems to country ballads, from hip-hop bangers to R&B slow jams. Your lyrics win Grammys. Your hooks are unforgettable. Your verses tell stories that move people to tears or make them dance.

You will receive a SONG BLUEPRINT containing the genre, mood, concept, vocal style, and musical direction. Your job is to write lyrics that are:

1. AUTHENTIC to the genre — use the vocabulary, cadence, and cultural references that belong to this style of music.
2. STRUCTURED with clear section markers: [Intro], [Verse 1], [Pre-Chorus], [Chorus], [Verse 2], [Bridge], [Chorus], [Outro]. Not every song needs every section — choose the structure that serves the song.
3. SINGABLE — lyrics must flow naturally when sung. Pay attention to syllable counts per line, vowel sounds on held notes, and rhythmic phrasing that matches the BPM.
4. EMOTIONALLY RESONANT — every line should serve the emotional arc. Verses build, pre-choruses create tension, choruses release and deliver the hook.
5. MEMORABLE — the chorus hook should be instantly singable after one listen. Use repetition strategically. Create earworms.
6. SOPHISTICATED — avoid clichés unless subverting them. Use vivid imagery, unexpected metaphors, and specific details that make the song feel lived-in and real.
7. GENRE-APPROPRIATE in length — Chorus: 4-8 lines. Verses: 8-16 lines. Bridge: 4-8 lines. Pre-Chorus: 2-4 lines.

FORMATTING RULES:
- Each section starts with a marker on its own line: [Verse 1], [Chorus], etc.
- One line of lyrics per line of text.
- Leave a blank line between sections.
- Do NOT include chords, timing marks, or production notes — only lyrics and section markers.
- If the song calls for ad-libs, write them in parentheses: (yeah, yeah)
- If the song has a spoken word intro or outro, mark it as [Spoken Intro] or [Spoken Outro].

OUTPUT: Only the lyrics with section markers. No commentary, no explanations.
```

#### Stage 3: Lyrics Quality Critic (Self-Refinement Loop)

```
SYSTEM PROMPT — LYRICS QUALITY CRITIC

You are the harshest, most respected music critic in the industry. You've reviewed thousands of songs and you know exactly what separates a good song from a chart-topping masterpiece.

You will receive LYRICS and a SONG BLUEPRINT. Score the lyrics on these criteria (1-10 each):

1. HOOK STRENGTH — Is the chorus instantly memorable? Would someone hum it after one listen?
2. RHYME SCHEME — Are rhymes natural and varied? No forced rhymes. Internal rhymes earn bonus points.
3. EMOTIONAL ARC — Does the song build and release tension? Does it take the listener on a journey?
4. SYLLABLE FLOW — Do the lyrics sit naturally on the rhythm? Would they be easy to sing at the given BPM?
5. GENRE AUTHENTICITY — Do these lyrics belong in this genre? Would fans of this genre embrace them?
6. IMAGERY & SPECIFICITY — Are the lyrics vivid and specific, or vague and generic?
7. ORIGINALITY — Does this feel fresh? Or does it recycle tired tropes?
8. SINGABILITY — Are the vowel sounds on key words open and resonant? Are held notes on strong syllables?

If ANY score is below 7, output REVISED LYRICS that address the weaknesses.
If ALL scores are 7+, output the lyrics unchanged with a note: "APPROVED — no revisions needed."

OUTPUT FORMAT:
{
  "scores": { "hook": N, "rhyme": N, "arc": N, "flow": N, "authenticity": N, "imagery": N, "originality": N, "singability": N },
  "overall": N (average),
  "weaknesses": ["list of specific issues"],
  "revised_lyrics": "full revised lyrics OR 'APPROVED — no revisions needed'"
}
```

## 6. ALBUM ARTWORK GENERATION PIPELINE

### 6.1 Artwork Prompt Engineering

The artwork prompt is derived from the song blueprint. It should capture the song's emotional essence in a vivid visual scene.

```javascript
// Workers AI — Artwork Generation Worker

async function generateArtwork(env, songBlueprint, songTitle) {
  // Step 1: Generate an art-direction prompt from the song metadata
  const artDirectionPrompt = await env.AI.run(LYRICS_MODEL_PRIMARY, {
    messages: [
      {
        role: "system",
        content: `You are an elite album cover art director. You create visual concepts for album artwork that capture the emotional essence of music. Your covers are iconic — think Childish Gambino's "Awaken My Love", Kendrick Lamar's "DAMN.", Burna Boy's "African Giant", or Adele's "30".

Given a song's metadata, output a single detailed image generation prompt (100-150 words) that describes an album cover. 

RULES:
1. NEVER include text, letters, words, or typography in the image description — text renders poorly in AI images.
2. Focus on mood, atmosphere, color palette, composition, and symbolic imagery.
3. Match the genre's visual language — Afrobeats covers use vibrant colors and African motifs; R&B uses intimate/moody lighting; Hip-hop uses bold graphic compositions; Pop uses bright, clean aesthetics.
4. Specify the art style: photographic, illustrated, abstract, collage, minimalist, surrealist, etc.
5. Include specific color palette guidance.
6. The image should work as a square (1:1 ratio) album cover.

OUTPUT: Only the image prompt. No commentary.`
      },
      {
        role: "user",
        content: `Song Title: "${songTitle}"
Genre: ${songBlueprint.genre} / ${songBlueprint.sub_genre}
Mood: ${songBlueprint.mood}
Artwork Direction: ${songBlueprint.artwork_mood}
Key Instruments: ${songBlueprint.instruments.join(", ")}
Concept: ${songBlueprint.song_concept}`
      }
    ],
    max_tokens: 300,
    temperature: 0.9
  });

  const imagePrompt = artDirectionPrompt.response;

  // Step 2: Generate the artwork using FLUX.2 or Leonardo
  const artworkResponse = await env.AI.run(ARTWORK_MODEL_PRIMARY, {
    prompt: imagePrompt,
    width: 1024,
    height: 1024,
    num_steps: 20,        // FLUX.2: 20-30 steps for quality
    guidance: 7.5,        // How closely to follow the prompt
  });

  // artworkResponse is a ReadableStream of image bytes (PNG)
  // Step 3: Upload to R2
  const artworkKey = `artwork/${songId}/cover.png`;
  await env.R2_BUCKET.put(artworkKey, artworkResponse, {
    httpMetadata: { contentType: "image/png" },
    customMetadata: { prompt: imagePrompt, model: ARTWORK_MODEL_PRIMARY }
  });

  return {
    artwork_url: artworkKey,
    artwork_prompt: imagePrompt
  };
}
```

### 6.2 Artwork Styles by Genre

The art director LLM should be aware of genre-appropriate visual styles:

```

Afrobeats/Afro-fusion:
```

## 8. DURABLE OBJECT — SONG GENERATION SESSION

```javascript
// Song generation Durable Object maintains state and pushes WebSocket updates

export class SongGenerationSession {
  constructor(state, env) {
    this.state = state;
    this.env = env;
    this.sessions = []; // Active WebSocket connections
  }

  async fetch(request) {
    const url = new URL(request.url);

    if (request.headers.get("Upgrade") === "websocket") {
      // Handle WebSocket connection for real-time updates
      const [client, server] = Object.values(new WebSocketPair());
      this.sessions.push(server);
      server.accept();
      server.addEventListener("close", () => {
        this.sessions = this.sessions.filter(s => s !== server);
      });

      // Send current state immediately
      const currentState = await this.state.storage.get("generationState");
      if (currentState) {
        server.send(JSON.stringify(currentState));
      }

      return new Response(null, { status: 101, webSocket: client });
    }

    if (url.pathname === "/start" && request.method === "POST") {
      const { songId, userPrompt, userId } = await request.json();
      return await this.startGeneration(songId, userPrompt, userId);
    }

    return new Response("Not found", { status: 404 });
  }

  broadcast(message) {
    const data = JSON.stringify(message);
    for (const session of this.sessions) {
      try { session.send(data); } catch (e) { /* Connection closed */ }
    }
  }

  async updateStage(stage, status, data = {}) {
    const state = {
      stage,
      status, // 'in_progress', 'completed', 'failed'
      timestamp: Date.now(),
      ...data
    };
    await this.state.storage.put("generationState", state);
    this.broadcast(state);
  }

  async startGeneration(songId, userPrompt, userId) {
    try {
      // STAGE 1: Blueprint
      await this.updateStage("blueprint", "in_progress");
      const blueprint = await this.generateBlueprint(userPrompt);
      await this.updateStage("blueprint", "completed", { blueprint });

      // STAGE 2: Lyrics
      await this.updateStage("lyrics", "in_progress");
      const lyrics = await this.generateLyrics(blueprint);
      await this.updateStage("lyrics", "completed", { lyrics_preview: lyrics.substring(0, 200) });

      // STAGE 3: Lyrics Refinement
      await this.updateStage("refinement", "in_progress");
      const refinedLyrics = await this.refineLyrics(lyrics, blueprint);
      await this.updateStage("refinement", "completed");

      // STAGE 4: Music Generation (ACE-Step — external GPU)
      await this.updateStage("music", "in_progress", { message: "Generating music variations..." });
      const audioFiles = await this.generateMusic(blueprint, refinedLyrics);
      await this.updateStage("music", "completed", { variation_count: audioFiles.length });

      // STAGE 5: Album Artwork
      await this.updateStage("artwork", "in_progress");
      const artwork = await this.generateArtwork(blueprint, songId);
      await this.updateStage("artwork", "completed", { artwork_url: artwork.artwork_url });

      // STAGE 6: Post-Processing
      await this.updateStage("processing", "in_progress");
      await this.postProcess(songId, userId, blueprint, refinedLyrics, audioFiles, artwork);
      await this.updateStage("processing", "completed");

      // STAGE 7: Done
      await this.updateStage("completed", "completed", { song_id: songId });

      return new Response(JSON.stringify({ success: true, songId }));
    } catch (error) {
      await this.updateStage("error", "failed", { error: error.message });
      return new Response(JSON.stringify({ success: false, error: error.message }), { status: 500 });
    }
  }

  // ... implement each stage method using the patterns from Section 5 and 6
}
```

## 13. SECURITY CONSIDERATIONS

- All API routes behind JWT authentication (except public explore endpoints)
- Rate limiting via Workers middleware (per-user, per-IP)
- Content safety: Run all user prompts through `@cf/meta/llama-guard-3-8b` before generation
- ACE-Step server: Nginx reverse proxy with API key validation, IP whitelist (only Cloudflare Worker IPs)
- R2 signed URLs for audio delivery (prevent hotlinking)
- CORS restricted to `melodia.pages.dev` and allowed origins
- Input sanitization on all user-provided text (lyrics edits, prompts)
- Audio watermarking for free-tier outputs (metadata-level, not audible)
